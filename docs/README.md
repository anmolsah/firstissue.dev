## Accessing the Documentation

Visit the documentation at:

- Main docs page: `/docs`
- Specific section: `/docs/{section-id}`
- Specific article: `/docs/{section-id}/{article-id}`

## Features

- **Responsive Design** - Works on all devices
- **Search Functionality** - Find articles quickly
- **Navigation Sidebar** - Easy browsing between sections
- **Code Examples** - Copy-paste ready code snippets
- **Progress Tracking** - Difficulty levels and read times
- **Modern UI** - Clean, dark theme consistent with the site

## Content Focus

The documentation is specifically designed for FirstIssue.dev's core mission - helping students and beginners discover and contribute to open source projects through our web platform. All content focuses on:

- Using the FirstIssue.dev website effectively
- Understanding open source contribution basics
- Finding and evaluating beginner-friendly issues
- Following proper contribution workflows
- Building skills and portfolio through contributions

## Content Management

Documentation content is managed in `src/data/docContent.js` with a structured format that supports:

- Headings (H2, H3, H4)
- Paragraphs
- Code blocks with syntax highlighting
- Ordered and unordered lists
- Callout boxes (info, warning, success, error)
- Metadata (read time, difficulty, last updated)

## Contributing to Documentation

To add or update documentation:

1. Edit `src/data/docContent.js`
2. Follow the existing content structure
3. Test your changes locally
4. Submit a pull request

The documentation system is designed to be easily maintainable and expandable as FirstIssue.dev grows.
